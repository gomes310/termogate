# 🚀 Deploy no GitHub Pages\n\n## Opção 1: Deploy Automático (Recomendado)\n\n### Passo 1: Configurar GitHub Pages\n\n1. Vá para **Settings** do repositório\n2. Procure por **Pages** no menu lateral\n3. Em **Source**, selecione:\n   - **Deploy from a branch**\n   - Branch: **gh-pages**\n   - Folder: **/ (root)**\n4. Clique em **Save**\n\n### Passo 2: O workflow já está configurado\n\nO arquivo `.github/workflows/deploy.yml` já foi criado e irá:\n- ✅ Fazer build automático ao fazer push para `main`\n- ✅ Fazer deploy automático no GitHub Pages\n- ✅ Gerar site em: `https://gomes310.github.io/Termogate/`\n\n### Passo 3: Fazer push\n\n```bash\ngit add .\ngit commit -m \"Add GitHub Pages deployment\"\ngit push origin main\n```\n\nProto! 🎉 O GitHub Actions irá:\n1. Fazer o build do projeto\n2. Fazer deploy automático\n3. Seu site estará disponível em:\n   **https://gomes310.github.io/Termogate/**\n\n---\n\n## Opção 2: Deploy Manual\n\nSe preferir fazer deploy manualmente:\n\n```bash\n# 1. Instalar dependências\nnpm install\n\n# 2. Fazer build\nnpm run build\n\n# 3. Deploy com gh-pages\nnpm run deploy\n```\n\nOu se não tiver gh-pages instalado:\n\n```bash\nnpm install --save-dev gh-pages\nnpm run deploy\n```\n\n---\n\n## Opção 3: Com Netlify (Alternativa)\n\n### Passo 1: Conectar ao Netlify\n\n```bash\nnpm install -g netlify-cli\nnetlify login\n```\n\n### Passo 2: Deploy\n\n```bash\nnetlify deploy --prod --dir=dist\n```\n\n---\n\n## Opção 4: Com Vercel\n\n### Passo 1: Instalar Vercel CLI\n\n```bash\nnpm install -g vercel\n```\n\n### Passo 2: Deploy\n\n```bash\nvercel --prod\n```\n\n---\n\n## 📊 Configurações Importantes\n\n### Vite Config\n\nJá está configurado em `vite.config.js`:\n\n```javascript\nbase: process.env.NODE_ENV === 'production' ? '/Termogate/' : '/'\n```\n\nIsso garante que os assets sejam carregados corretamente no caminho `/Termogate/`\n\n### Environment Variables\n\nPara o GitHub Pages, atualize a URL da API em `.env`:\n\n```env\nVITE_API_URL=http://localhost:5000\n```\n\nPara produção (sem backend real), use mock data ou um servidor específico.\n\n---\n\n## ✅ Checklist de Deploy\n\n- [ ] GitHub Pages ativado em Settings\n- [ ] `.github/workflows/deploy.yml` criado\n- [ ] `vite.config.js` configurado com `base`\n- [ ] `package.json` com `homepage`\n- [ ] Push feito para `main`\n- [ ] GitHub Actions iniciou o build\n- [ ] Site disponível em GitHub Pages\n\n---\n\n## 🔗 URLs Finais\n\nApós o deploy:\n\n**GitHub Pages:**\n```\nhttps://gomes310.github.io/Termogate/\n```\n\n**Custom Domain (opcional):**\n```\nhttps://termogate.gomes310.dev/\n```\n\n---\n\n## 📱 Como Usar a Página Publicada\n\n1.
